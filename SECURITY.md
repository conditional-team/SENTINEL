## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue, please report it responsibly.

### How to Report

1. **DO NOT** open a public GitHub issue for security vulnerabilities
2. Email us at: **security@sentinel-shield.io** (replace with actual email)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes

### What to Expect

- **Acknowledgment**: Within 24 hours
- **Initial Assessment**: Within 72 hours
- **Resolution Timeline**: Depends on severity
  - Critical: 24-48 hours
  - High: 1 week
  - Medium: 2 weeks
  - Low: 1 month

### Bug Bounty

We offer rewards for responsibly disclosed vulnerabilities:

| Severity | Reward |
|----------|--------|
| Critical | Up to $10,000 |
| High     | Up to $5,000 |
| Medium   | Up to $1,000 |
| Low      | Up to $250 |

### Scope

In scope:
- Smart contracts (SentinelRegistry, BatchRevokeYul)
- API server vulnerabilities
- Frontend security issues
- Authentication/authorization bypasses

Out of scope:
- Social engineering
- Physical attacks
- DoS attacks on infrastructure
- Issues in third-party dependencies (report to them directly)

## Security Best Practices

When using SENTINEL SHIELD:

1. **Never share your private keys**
2. Always verify contract addresses before interacting
3. Review approval transactions carefully
4. Keep your wallet software updated
5. Use hardware wallets for significant holdings
